Project Circuit

A new software platform that brings together text, voice, video, conferencing, and enterprise social networking in a seamless and intuitive experience.

Project Circuit was one of the largest projects I worked on during my time at frog. A multiplatform enterprise communication tool, working seamlessy across each device for private and public communication.

The project ran for more than 18 months and I was early in the project with the main task of defining the main interaction model for the communication within the tool. Later in the project I was also tasked to translate the same model to be adapted to pretty old-school devices like desk phones.

Defining a conversation model for a tool that has infinite users configuration and infinite potential usecases is a pretty complex task. Since the start I had to find my own ways to explain possible interaction models with quick sketches and animations.

Big part of the task was to figure out how to communicate and detail all the design decisions to the development team, using a mix of flows, animations and quick prototype.

To test scenarios that needed a multi user interaction, like conference calls and live messaging, I created quite hacky networked prototypesusing Processing and Spacebrew to simulate a communicate between devices
(at the time I had no clue of Nodejs...)

In a second part of the project we had to consider more complex scenarios, devices and edgecases. One of my main tasks was to translate the same interaction modelto a non-touch screen desk phone and considering the complexities of multi-line call systems.

We had to convince the client that a new digital interface needs a new physical configuration of buttons. So I ended up building a mock up phone, with a lot of wires and spaghetti code.
